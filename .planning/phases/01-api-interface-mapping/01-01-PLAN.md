## Interfaces

From PDF (0x03 API.获取接口授权码):
- Endpoint: `GET /v3/token`
- Request params: `appid` (String, required), `secret` (String, required)
- Response: `{ message: "ok", token: String, expires: Integer }` — expires in seconds (7200)
- Example: `GET http://openapi.ecois.info/v3/token?appid=test&secret=test`

From EcoApiAuth.java (DB model):
- Fields: id, name, token, uid, ipAllow, debug, createTime, updateTime, appid, appsecret, notify_url, lastModifyTime

From EcoApiAuthService.java:
- `getAuthByAppId(String appid, String secret)` — validates appid+secret, returns auth record with token
- `getAuthByToken(String token)` — validates existing token
- `getAuthByUid(Integer uid)` — lookup by user id

From CONTEXT.md decisions:
- D-10: `authenticate` tool with appid + secret → token
- D-11: Token expiry detection + auto-refresh in skill.md prompt
- D-12: Conversation-level token reuse, cache in context, refresh on 401/403
</context>
