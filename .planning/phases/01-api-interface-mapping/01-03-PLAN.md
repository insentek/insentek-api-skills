3. **Skill Tool Definitions** (one section per tool):

   **3.1 authenticate**
   - Purpose: Obtain and manage API access token
   - Mapped endpoints: `GET /v3/token`
   - Parameters: appid (String, required), secret (String, required)
   - Returns: token (String), expires (Integer, seconds)
   - Behavior:
     - Call /v3/token with appid + secret
     - Cache token in conversation context
     - Reuse for all subsequent API calls via `Authorization: {token}` header
     - Refresh when 401/403 received or expires approaching
   - Error handling: 401 → "认证失败，请检查 appid 和 secret"; 500 → retry then fail
   - References: D-10, D-11, D-12, AUTH-01, AUTH-02

   **3.2 query_device**
   - Purpose: Query device information (list, detail, parameters)
   - Mapped endpoints:
     - `GET /v3/devices` — device list with pagination
     - `GET /v3/device/{sn}` — single device detail
     - `GET /v3/device/{sn}/description` — parameter descriptions
   - Parameters:
     - page (Integer, optional, default 1)
     - limit (Integer, optional, default 20)
     - sn (String, optional — if provided, skip list and go to detail)
     - alias (String, optional — if provided, call /v3/devices to resolve sn first)
   - Returns: Device list or single device detail with parameter meanings
   - Behavior:
     - If alias provided → call /v3/devices → find matching sn → call /v3/device/{sn} + /v3/device/{sn}/description
     - If sn provided → call /v3/device/{sn} + /v3/device/{sn}/description
     - If neither → call /v3/devices with page/limit
     - Use /v3/device/{sn}/description to enrich response with parameter meanings
   - Chain pattern: Often called before `query_data` to resolve sn from alias
   - References: D-04, D-06, DEV-01, DEV-02

   **3.3 query_data**
   - Purpose: Query device data (historical, real-time, moment, incremental)
   - Mapped endpoints:
     - `GET /v3/device/{sn}/data` — historical data with date range
     - `GET /v3/device/{sn}/latest` — latest real-time data
     - `GET /v3/device/{sn}/data/moment/{datetime}` — data at specific moment
     - `GET /v3/device/{sn}/data/incremental` — incremental sync data
   - Parameters:
     - sn (String, required)
     - time_expression (String, optional — natural language time reference)
     - range (String, optional — "YYYYMMDD,YYYYMMDD", auto-calculated from time_expression)
     - includeParameters (String, optional — comma-separated parameter filters)
   - Returns: Data points with timestamp, datetime, and values
   - Auto-inference rules (per D-05, D-07, D-08):
     - "现在/最新/当前/实时" → /latest
     - "昨天/上周/最近N天/本月/本周/今年" → /data with calculated range
     - "某个时间点/某时刻" → /moment/{datetime}
     - "同步/增量/更新" → /incremental
     - No time clue → /data with last 24h range (default)
   - Date calculation rules:
     - "最近7天" → today-7d to today
     - "昨天" → yesterday to yesterday
     - "本周" → Monday to today
     - "本月" → 1st to today
     - Format as "YYYYMMDD,YYYYMMDD"
   - Chain pattern: Requires sn — if user provides alias, call `query_device` first
   - References: D-04, D-05, D-06, D-07, D-08, D-09, DATA-01, DATA-02

   **3.4 config_webhook** (Optional)
   - Purpose: Configure data push webhook URL
   - Mapped endpoints: `GET /v3/device/webhook` (query), `POST /v3/device/webhook` (set)
   - Parameters: notifyUrl (URL, required for set)
   - Returns: Current webhook URL and last modified time
   - Mark as "Optional — advanced configuration, not core data query"
   - Reference: D-02

4. **Interaction Flow Diagrams** (text-based):

   - Flow 2: "查询某设备最新数据" → authenticate → query_device (resolve sn) → query_data (/latest)

   - Flow 4: "查询某时刻数据" → authenticate → query_device (resolve sn) → query_data (/moment)
