## Interfaces

### Device APIs (from PDF)

**GET /v3/devices** — 设备列表
- Params: `page` (Integer, required), `limit` (Integer, required)
- Response: `{ count: Integer, list: Array<DeviceSummary> }`
- DeviceSummary fields: sn, alias, type_id/type, series, location (lng, lat, country, province, city, district), authorized (own/shared), status (code, description), widget (externalAccessUrl)

**GET /v3/device/{sn}** — 单设备详情
- Params: `sn` (path, String, required)
- Response: DeviceDetail = DeviceSummary + authorizedUsers (openid, name, headimg) + feature (online, rssi, network) + widget

**GET /v3/device/{sn}/attr** — 修改设备属性（per PDF 0x17, POST）
- Actually: `POST /v3/device/{sn}/attr` — 修改设备名称和上报周期
- Body: `{ name: String, reportCycle: Integer }` — reportCycle values: 5, 10, 20, 30, 60, 120 (minutes)
- Response: `{ message: "ok" }`
- Note: For skill.md, this is read-only context (device attr query is via /v3/device/{sn})

**GET /v3/device/{sn}/description** — 设备参数说明
- Response: `{ nodes: Array<String>, parameters: Array<Object> }`
- nodes and parameters correspond by index
- parameter object: `{ name: String, unit: String, type: Enum(text/numeric/binary) }`
- Example: nodes=["地表","10cm","20cm","30cm","40cm"], parameters=[{moisture:{name:"含水量",unit:"%",type:"numeric"}, temperature:{name:"温度",unit:"℃"}}]

### Data Query APIs (from PDF)

**GET /v3/device/{sn}/data** — 历史数据
- Params: `range` (String, optional, format "YYYYMMDD,YYYYMMDD"), `includeParameters` (String, optional, comma-separated keys), `includeNodes` (String, optional)
- Response: `{ total: Integer, list: Array<DataPoint> }`
- DataPoint: `{ timestamp: Timestamp, datetime: DateTime, values: Object }`
- values structure: `{ "10cm": { moisture: 12.3, temperature: 10.7 }, ... }`

**GET /v3/device/{sn}/latest** — 最新实时数据
- Params: `sn` (path, required)
- Response: `{ message: "ok", timestamp, datetime, lng, lat, values: Object }` — same values structure as /data

**GET /v3/device/{sn}/data/moment/{datetime}** — 指定时刻数据
- Params: `sn` (path), `datetime` (path, DateTime, URL-encoded)
- Response: `{ message: "ok", timestamp, datetime, values: Object }`

**GET /v3/device/{sn}/data/incremental** — 增量数据
- Params: `sn` (path)
- Response: `{ message: "ok", lastSync: DateTime, increments: Array<IncrementalDataPoint> }`
- IncrementalDataPoint: `{ timestamp, datetime, values: Object }`
- First call returns last 3 months; subsequent calls return data since lastSync

### Device Status Codes (from PDF 0x17)
- `Used`: 土壤墒情仪、气象站的工作中状态
- `Fault`: 土壤墒情仪、气象站的故障中状态
- `Online`: 云衍、聆耘的在线状态
- `Offline`: 云衍、聆耘的离线状态
- `Produce`: 生产中
- `ToBeDelivered`: 待发货
- `Repair`: 维修中
- `Idle`: 未部署
- `Scrap`: 已报废

### Parameter Names by Device Type (from PDF 0x17)
- Z (土壤监测设备): temperature, moisture, ec, battery, outsideVoltage
- T (气象监测设备): airTemperature, relativeHumidity, dewPoint, atmosphericPressure, maxWindSpeed, averageWindSpeed, windDirection, rainfall, rainfallDuration, solarRadiationIntensity, solarRadiationAmount, sunshineDuration, pm1.0, pm2.5, pm10, tvoc
- J (见厘液位计): laserliquidLevel, battery, outsideVoltage

### Context Decisions
- D-04: `query_device` aggregates /v3/devices + /v3/device/{sn} + /v3/device/{sn}/attr (read) + /v3/device/{sn}/description
- D-04: `query_data` aggregates /v3/device/{sn}/data + /v3/device/{sn}/latest + /v3/device/{sn}/data/moment/{datetime} + /v3/device/{sn}/data/incremental
- D-05: Agent auto-infers query type from time clues in natural language
- D-07: Natural language time parsing ("最近7天" → auto-calculate date range)
- D-08: Default to last 24h when no time range specified
- D-09: `range` parameter format is "YYYYMMDD,YYYYMMDD"
